School Management System


App created to solve the problem of disorganization in schools by having to assign a classroom or check if there is one available.

This project was designed with a continuous release methodology in order to constantly improve the application and carry out tests with possible users.


Features

The app is designed so that one single user can manage write operations like editing, adding or deleting an entry. Any other user will only be able to read them.

For both scenarios, one must first create an account.

The user who can control the write operations must have some kind of identification so that the app knows whether the user can access specific endpoints or not.

By default, the app will expect the user who can access these resources to have an email with a specific prefix.

Details

To build this application, I used the Spring Boot framework for an easy setup.
I also used the Spring Security module to add authentication and authorization mechanisms.
To maintain user data and classroom table entries, I used MySQL database together with JPA.
And for the frontend I used Thymeleaf to create a user-friendly interface for the CRUD operations.

Challenges

This was one of the first backend projects I developed using Java, so there were a lot of things, like frameworks and modules, that I did not know anything about.

Security and authorization were the main problems I had to deal with, establishing what a user can and cannot access and how to implement this was both as problematic as challenging, and ended up being fun.

But in the end, through documentation and online tutorials I was able to finish create this application that I really enjoyed developing.

What's next

  • Deciding how a user can have access to write operations(it could be by username instead of email).
  • New tables related to the classroom table can be added so that more information can be seen.
